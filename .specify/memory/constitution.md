# Pricing Optimizer API Constitution

## Core Principles

### I. Contract-First (OpenAPI 3.1)

The `openapi.yaml` document is the single source of truth for the HTTP contract. It is
authored and owned at the umbrella root and synced into this repo via `make sync-openapi`;
this repo's copy MUST NOT be edited directly. No route, request schema, or response schema
ships without the spec being updated first. Handlers and types are generated from the spec
with oapi-codegen and MUST NOT be hand-edited where generation is authoritative. The spec
stays on OpenAPI 3.1 per ADR-0004; authored constructs MUST remain within the generator's
supported subset.

Rationale: A generated, spec-driven contract eliminates drift between server, client, and
docs, and makes the contract reviewable independently of implementation.

### II. Clean Architecture Layer Boundaries

The codebase follows a pragmatic four-layer Clean Architecture with strictly enforced
import direction:
- `domain` imports nothing (pure entities and ports/interfaces).
- `usecase` imports `domain` only — never adapters, never external packages.
- `adapter` imports `domain` — never `usecase`.
- `cmd` imports everything and wires dependencies at startup.

Business logic MUST live in `usecase`/`domain`, isolated from HTTP, LLM, DB, and cache
concerns behind ports. Any violation of import direction is a blocking review failure.

Rationale: Isolating business logic behind ports keeps it testable without infrastructure
and lets adapters (LLM provider, DB, scraper) change without touching core logic.

### III. Test-First Rigor (NON-NEGOTIABLE)

Tests are written to accompany or precede implementation, never as an afterthought.
- Tests MUST be table-driven, using `t.Run` with descriptive English names; helpers call
  `t.Helper()`.
- Mocks MUST be generated by uber-go/mock — never hand-written.
- Integration tests MUST use testcontainers-go against a real Postgres — no DB mocks.
- LLM adapter tests MUST use a mocked client plus golden fixtures.
- Coverage MUST be >= 80% in `internal/usecase` and `internal/domain`.

Rationale: The senior-signaling value of this project depends on demonstrable correctness;
generated mocks and real-dependency integration tests prevent false confidence.

### IV. LLM Provider Abstraction & Structured Output

All LLM access goes through a single `LLMProvider` port selected by an env-based factory.
No use case or handler may know which provider is active. Generation MUST use structured
tool calling with a JSON schema; parsing free-form model text is prohibited. Provider
selection: Groq (`llama-3.3-70b-versatile`) in production, Anthropic (Claude Sonnet) in
development, Groq 8B as throttling fallback (ADR-0003).

Rationale: Structured output removes a whole class of runtime parsing errors, and the port
boundary keeps provider choice a config decision rather than a code change.

### V. Observability From Day One

Observability is wired in the first sprint, not deferred. OpenTelemetry traces MUST cover
every HTTP handler, DB call, and LLM call; every LLM call MUST be its own span. Logs MUST
be structured JSON via `log/slog` and MUST propagate `trace_id`. An `llm_calls` record MUST
be persisted for cost and latency analysis. `fmt.Println` is prohibited in shipped code.

Rationale: Streaming and LLM systems are opaque without tracing; building it in from the
start is both a reliability and a portfolio requirement.

### VI. Shipped-Artifact Discipline

All shipped artifacts — code, comments, commit messages, PR titles and bodies, READMEs,
ADRs, and issues — MUST be in English. Commits MUST follow Conventional Commits, enforced
by commitlint + lefthook. No emojis in commits, PR titles, or code. Secrets MUST come from
environment variables only, never hardcoded. Every non-trivial architectural decision made
while coding MUST be recorded as an ADR in `../docs/decisions/`.

Rationale: This is a portfolio piece for international reviewers; consistency and hygiene
are themselves signals of engineering care.

## Additional Constraints (Stack & Cost)

- Language and core stack are fixed: Go 1.23+, go-chi/chi v5, oapi-codegen, chromedp +
  colly, pgx v5 + sqlc, pressly/goose, redis/go-redis v9, OpenTelemetry SDK, testify +
  testcontainers-go + uber-go/mock, golangci-lint, air.
- Streaming for `/v1/generate` MUST use Server-Sent Events, not WebSockets.
- The three variations MUST be generated in parallel (goroutines + errgroup).
- Ongoing infrastructure cost target is $0/month; no paid infrastructure may be introduced.
  Production runs on free tiers (Fly.io, Neon, Upstash, Groq, Grafana Cloud, Sentry).
- The final container image MUST be a multi-stage, distroless build under 30MB.

## Development Workflow & Quality Gates

- Spec-driven workflow: `/speckit-constitution` → `/speckit-specify` → `/speckit-plan` →
  `/speckit-tasks` → `/speckit-implement`. Structure precedes code.
- Branch-per-task (NON-NEGOTIABLE): `main` is protected and MUST NOT receive direct
  commits. A dedicated branch is created BEFORE any development on a task begins, from an
  up-to-date `main`. Branch naming:
  - Spec-driven feature: `NNN-slug` created by `/speckit-specify` (e.g. `002-generate-variations`).
    Do not rename it — the `specs/NNN-slug/` directory must match.
  - Standalone change (tooling, CI, deps, docs, hotfix without a full spec):
    `<type>/<slug>` where `<type>` is a Conventional Commit type (feat, fix, chore,
    refactor, docs, ci, build, perf, test) and `<slug>` is kebab-case.
  - One branch = one logical unit of work. Keep branches short-lived.
- Every change reaches `main` only through a Pull Request. A PR MUST NOT be merged until it
  is approved: `pr-reviewer` review addressed, CI green, and all quality gates below passing.
  Delete the branch after merge.
- Every PR MUST pass: zero golangci-lint warnings, the full test suite, coverage gates on
  `usecase`/`domain`, and `make check-openapi` (contract in sync with the root).
- lefthook pre-commit and pre-push hooks MUST be active; CI MUST be green on `main`.
- A PR that changes the contract MUST update the root `openapi.yaml`, run `make
  sync-openapi`, and regenerate code in the same change.

## Governance

This constitution supersedes ad-hoc practices for this repository. Amendments MUST be made
by editing this file with a Sync Impact Report and a semantic version bump: MAJOR for
backward-incompatible principle removals or redefinitions, MINOR for added or materially
expanded principles/sections, PATCH for clarifications. Dependent templates
(`plan-template.md`, `spec-template.md`, `tasks-template.md`) MUST be reviewed for
alignment on every amendment. All PRs and reviews MUST verify compliance with these
principles; added complexity MUST be justified against them. Runtime development guidance
lives in `CLAUDE.md` and the umbrella `../.claude/CLAUDE.md`.
